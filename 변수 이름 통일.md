# lobby.js와 room.js 변수명 통일 가이드

`server.js`와 WebRTC 이벤트를 연결하기 전에 `lobby.js`와 `room.js`의 변수명을 먼저 통일하기 위한 문서이다.

## 기본 규칙

JavaScript 변수와 함수 이름은 `camelCase`로 통일한다.

```javascript
localVideo
remoteVideo
localStream
remoteStream
localMicButton
localScreenButton
roomTitle
roomId
```

DOM 요소도 대문자로 시작하지 않고 소문자로 시작한다. 대문자로 시작하는 이름은 일반적으로 클래스 이름처럼 보일 수 있다.

### `roomtitle`을 `title`로 변경

기존 코드:

```javascript
const roomtitle = "방 이름 정해야 됨"
```

권장 코드:

```javascript
const title = prompt("방 제목을 입력하세요")
```

서버에서도 `req.body.title`을 사용하므로 요청 데이터 역시 `title`로 맞춘다.

```javascript

```

### `lobbyData`를 `rooms`로 변경

기존 코드:

```javascript

renderLobby(lobbyData)
```

권장 코드:

```javascript
const rooms = await response.json()
renderLobby(rooms)
```

방 생성 응답은 단순한 `data`보다 `createdRoom`으로 지정하면 의미가 명확해진다.

```javascript
window.location.href =
    `/meetingRoom/room.html?roomId=${createdRoom.roomId}`
```

### `currentLobby()`를 `loadRooms()`로 변경

`currentLobby`보다 방 목록을 불러온다는 의미의 `loadRooms`가 자연스럽다.

## room.js

### DOM 요소 변수명 통일

기존 변수와 권장 변수는 다음과 같다.

| 기존 이름 | 권장 이름 | 의미 |
|---|---|---|
| `roomtitle` | `roomTitle` | 방 제목 DOM 요소 |
| `LocalId` | `localNickname` | 내 닉네임 DOM 요소 |
| `RemoteId` | `remoteNickname` | 상대 닉네임 DOM 요소 |
| `LocalMicBtn` | `localMicButton` | 내 마이크 버튼 |
| `LocalScreenBtn` | `localScreenButton` | 내 화면 공유 버튼 |
| `RemoteMicBtn` | `remoteMicButton` | 상대방 음소거용 버튼 |
| `RemoteScreenBtn` | `remoteScreenButton` | 상대 화면 공유 상태 버튼 |
| `LocalVideo` | `localVideo` | 내 카메라 video 요소 |
| `RemoteVideo` | `remoteVideo` | 상대 카메라 video 요소 |
| `LocalScreen` | `localScreenVideo` | 내 화면 공유 video 요소 |
| `RemoteScreen` | `remoteScreenVideo` | 상대 화면 공유 video 요소 |
| `localMicImg` | `localMicImage` | 마이크 버튼 내부 이미지 |
| `localStrem` | `localStream` | 내 미디어 스트림 |
| 선언되지 않은 `track` | `audioTrack` | 내 오디오 트랙 |
| 지역 변수 `screenTrack` | 상위 변수 `screenTrack` | 화면 공유 트랙 |
| `params` | `searchParams` | URL 쿼리 분석 객체 |
| `let roomId` | `const roomId` | URL에서 읽은 방 ID |

### 버튼 내부 이미지 변수

HTML에서 ID가 붙은 요소는 `<img>`가 아니라 `<button>`이다. 버튼의 `src`를 바꾸지 말고 버튼 내부 이미지를 가져와야 한다.

```javascript
const localMicImage = localMicButton.querySelector("img")
const localScreenImage = localScreenButton.querySelector("img")
```

이미지 변경 예시:

```javascript
localMicImage.src = isMicOn
    ? "./image/mic-on.png"
    : "./image/mic-off.png"
```

### 미디어 관련 변수 통일

```javascript
let localStream = null
let remoteStream = null
let peerConnection = null

let videoTrack = null
let audioTrack = null
let screenTrack = null

let isMicOn = true
```

`localStrem`은 오타이므로 모든 사용 위치를 `localStream`으로 변경한다.

```javascript
localStream = await navigator.mediaDevices.getUserMedia({
    video: true,
    audio: true
})

videoTrack = localStream.getVideoTracks()[0]
audioTrack = localStream.getAudioTracks()[0]
localVideo.srcObject = localStream
```

### DOM 요소와 MediaStream 구분

다음 두 변수는 서로 다른 역할이다.

```text
remoteVideo
→ HTML의 <video> 요소

remoteStream
→ 상대방의 영상과 음성 트랙을 담는 MediaStream 객체
```

따라서 다음처럼 사용한다.

```javascript
remoteStream = new MediaStream()
remoteVideo.srcObject = remoteStream
```

`remoteVideo = new MediaStream()`처럼 DOM 요소 변수에 스트림을 덮어쓰면 안 된다.

### 오디오 트랙 변수

기존 코드의 `track`은 선언되지 않은 변수이다.

```javascript
track.enabled = isMicOn
```

다음처럼 `audioTrack`으로 통일한다.

```javascript
audioTrack = localStream.getAudioTracks()[0]

isMicOn = !isMicOn
audioTrack.enabled = isMicOn
```

### 화면 공유 트랙 변수

화면 공유 함수 내부에서 `const screenTrack`으로 만들면 다른 함수에서 사용하기 어렵다. 상단에 선언한 변수에 값을 대입한다.

```javascript
let screenTrack = null
```

```javascript
screenTrack = screenStream.getVideoTracks()[0]
```

### URL 관련 변수

기존 코드:

```javascript
const params = new URLSearchParams(window.location.search)
let roomId = params.get("roomId")
```

권장 코드:

```javascript
const searchParams = new URLSearchParams(window.location.search)
const roomId = searchParams.get("roomId")
```

`roomId`는 URL에서 한 번 읽은 뒤 다시 대입하지 않으므로 `const`가 적합하다.

### 현재 제거 가능한 변수

현재 카메라 켜기·끄기 기능에서 사용하지 않는다면 다음 변수는 제거할 수 있다.

```javascript
let isCameraOn = true
```

`remoteMicButton`, `remoteScreenButton`, `remoteScreenVideo`도 아직 관련 기능을 구현하지 않았다면 선언만 남겨두거나 우선 제거할 수 있다.

상대방의 장치를 직접 켜거나 끌 수는 없다. `remoteMicButton`은 내 브라우저에서 상대방 소리를 음소거하는 용도로만 사용할 수 있다.

## 변경 순서

1. 선언부의 변수명을 권장 이름으로 변경한다.
2. 각 변수의 모든 사용 위치를 같은 이름으로 변경한다.
3. `localVideo`와 `remoteVideo`는 DOM 요소로만 사용한다.
4. `localStream`과 `remoteStream`은 `MediaStream` 객체로만 사용한다.
5. 이름 통일이 끝난 뒤 `createPeerConnection()`, Offer, Answer, ICE candidate 처리를 작성한다.
6. 마지막으로 클라이언트 이벤트 이름과 `server.js` 이벤트 이름을 맞춘다.

변수명을 바꿀 때는 선언부만 변경하지 말고 해당 변수의 모든 사용 위치를 함께 변경해야 한다.
